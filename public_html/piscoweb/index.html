· PISCO BOX ·

LAMP development environment based on Debian Bookworm with Apache, PHP 8.3 and MariaDB

Current Version: 0.2.0

Getting Started

To start your development environment with Pisco Box:

$ vagrant up

With the virtual machine running, you can log in:

$ vagrant ssh
Note: Make sure you have Vagrant and VirtualBox/Parallels/VMware installed before starting.

Web Access

The default site will be available at:

# Main URL
http://localhost:8080

# If you configured a local hostname
http://piscobox.test
How to configure a local hostname? follow these instructions to create a hostname on your local computer: by editing the /etc/hosts file

MySQL/MariaDB Credentials

User: piscoboxuser

Password: DevPassword123

Host: localhost

To connect to MariaDB from the command line:

# As root user
$ sudo mysql
# Or the equivalent
$ sudo mariadb

# As database user from command line:
$ mariadb -u piscoboxuser -p
# When prompted, enter the password: DevPassword123

Or from your PHP application:

$mysqli = new mysqli("localhost", "piscoboxuser", "DevPassword123", "piscoboxdb");

Main Features

  • Operating System: Debian Bookworm 64-bit
  • Tools: wget, curl, gnupg, lsb-release, ca-certificates, net-tools, git, vim
  • Time Zone: UTC
  • Locale Configuration: UTF-8
  • Web Server: Apache 2 with modules: proxy_fcgi, setenvif, rewrite
  • Web Directory: Configured and ready to use
  • PHP: 8.3 with PHP-FPM details
  • PHP Extensions: bcmath, bz2, cgi, gd, imap, intl, mbstring, pspell, tidy, xmlrpc, zip
  • Composer: Version 2.x
  • Database: MariaDB Server & Client

Project Structure

This is what the directory structure looks like on your local machine:

# Your project directory (local machine)
/path/to/your/piscobox
├── doc/ # Documentation and collaboration
├── extra_data/ # Extra data, backups
├── provision/ # Provisioning scripts
├── public_html/ # Document root accessible via web
├── utils/ # Workflow templates and others
├── LICENSE # Software License
├── README.md # Welcome, basic instructions
├── Vagrantfile # Pisco Box configuration
└── ... # Other project files

# Synchronized directory in the VM
/var/www/html ← Synchronized with public_html/
Important: Some directories like public_html/ and provision/files/ are synchronized with the virtual machine. This means that changes you make to files on your local machine will be immediately reflected in the virtual machine.

Useful Commands

# Restart Apache
$ sudo systemctl restart apache2

# Restart PHP-FPM
$ sudo systemctl restart php8.3-fpm

# View Apache logs
$ sudo tail -f /var/log/apache2/error.log
